## Philosophy

Every module answers three questions: **Why does this exist? How does it work internally? What breaks in production?**

Modules are never shallow. Every module includes first-principles explanations, real architecture diagrams, production failure scenarios, debuggable labs, interview Q&A chains, flashcards, and cheat sheets.

---

## Contributing / Self-Study Workflow

1. Read the module `README.md` top to bottom.
2. Run the lab scripts on your local machine.
3. Complete the exercises (attempt before reading answers).
4. Review flashcards until you can answer all 20 without prompting.
5. Answer the cross-question chain out loud as if in an interview.
6. Update `PROGRESS.md`.
